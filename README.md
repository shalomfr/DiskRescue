# DiskRescue — אבחון לפני שחזור

כלי שחזור מידע ל-Windows עם עיקרון מנחה אחד: **קודם אבחון מהיר, ורק אם צריך — שחזור מאסיבי.**

## ⬇️ הורדה

**[הורד את הגרסה האחרונה (DiskRescue.exe)](../../releases/latest)**

קובץ EXE יחיד ועצמאי — רץ על כל Windows 10/11 ללא התקנה. לחיצה כפולה → אישור UAC → הממשק נפתח.
(הכלי דורש הרשאות מנהל לגישה גולמית לדיסק, ולכן ייפתח תמיד עם בקשת UAC.)

רוב כלי השחזור קופצים ישר לסריקה עמוקה של שעות. DiskRescue קודם בודק את המבנה של הכונן (סקטור אתחול, גיבוי, דגל "מלוכלך", טבלת קבצים) — ולעיתים קרובות מגלה שמדובר בתיקון של חצי דקה, לא באסון.

## יכולות (גרסה נוכחית)

### 1. טריאז' (אבחון קריאה-בלבד)
- מצאי כוננים עם בריאות פיזית (WMI / MSFT_Disk / MSFT_Volume).
- קריאת סקטור אתחול והגיבוי שלו ישירות מהדיסק הפיזי (NTFS / exFAT / FAT32).
- זיהוי כונן RAW שסקטור האתחול שלו נמחק — והאם יש גיבוי תקין לשחזור.
- זיהוי דגל "מלוכלך" (exFAT VolumeFlags / NTFS dirty bit) → הסיבה ל-"Full Repair Needed".
- אימות שלמות `$MFT` ב-NTFS.
- עץ החלטה שמסכם: תיקון מהיר / chkdsk / שחזור עמוק / חשד לתקלה מכנית.

### 2. תיקונים בטוחים (כתיבה, עם ביטול)
- **שחזור סקטור אתחול מהגיבוי** — מעתיק את הגיבוי התקין לסקטור 0. שומר עותק לביטול, מאמת לפני ואחרי, מסרב לדרוס סקטור אתחול קיים.
- **chkdsk /f /x** — ניקוי דגל "מלוכלך" בכונן מזוהה ונגיש.

### 3. שחזור עמוק (Deep Scan) — חיתוך לפי חתימות
- סריקה רציפה של המחיצה וזיהוי קבצים לפי כותרות (jpg, png, gif, bmp, pdf, zip/docx, doc, mp4, mp3, wav).
- שחזור מאורגן לפי סוג קובץ לתיקיות.
- **השהיה / המשך / עצירה** באמצע סריקה.
- **שמירת פרויקט** (`.drproj`) והמשך אחר כך מאותה נקודה (checkpoint כל 256MB).
- עמידות לסקטורים פגומים — דילוג והמשך במקום קריסה.

### 4. שחזור מבוסס NTFS — דפדפן קבצים + Undelete
- פענוח ה-`$MFT`: רשימת כל הקבצים והתיקיות עם **הנתיב המקורי**.
- זיהוי **קבצים שנמחקו** (רשומות MFT שעדיין קיימות) לשחזור מהיר.
- שחזור נתונים: resident (קבצים קטנים) ו-non-resident (דרך data runs), כולל קבצים מקוטעים.
- שמירה על מבנה התיקיות המקורי בעת השחזור.
- ממשק: סריקה עם ביטול, סינון "נמחקים בלבד", חיפוש, ושחזור מרובה-בחירה.

### 5. שכפול דיסק (Imaging) + SMART — לכוננים גוססים
- קריאת **SMART**: בריאות, חיזוי כשל, טמפרטורה, שגיאות קריאה, בלאי, שעות פעולה.
- **שכפול בסגנון ddrescue**: מעבר ראשון מהיר חוטף את כל הקריא ומדלג על סקטורים פגומים; מעבר שני סורק את האזורים הפגומים בגרנולריות עדינה.
- **מפת התקדמות** (`.drmap`) — השהיה/המשך, ועצירה אוטומטית אם SMART מנבא כשל.
- לאחר השכפול: הרצת השחזור (דפדפן/עמוק) **על קובץ התמונה**, לא על הכונן המקורי.
- העיקרון: *קודם משכפלים, אחר כך משחזרים מהעותק.*

## בטיחות
- גישת קריאה כברירת מחדל. כל כתיבה שומרת עותק ביטול תחת `%LOCALAPPDATA%\DiskRescue\undo`.
- שחזור עמוק חוסם כתיבה אל אותו כונן שמשוחזר.
- דורש הרשאות מנהל (גישה גולמית לדיסק) — בקשת UAC אחת בהפעלה.

## בנייה והרצה
```
dotnet build -c Debug
# הפעלת הממשק (תבקש UAC):
bin\Debug\net9.0-windows\DiskRescue.exe
```

### מצבי CLI (לבדיקות)
```
# דוח טריאז' של כל הכוננים (דורש מנהל):
DiskRescue.exe --cli report.txt

# בדיקת מנוע החיתוך על קובץ סינתטי (ללא מנהל):
dotnet DiskRescue.dll --carvetest <out_dir> <report.txt>
```

## ארכיטקטורה
```
Core/
  Native.cs              P/Invoke לגישה גולמית (CreateFile/Read/Write)
  RawDevice.cs           קריאה/כתיבה לפי היסט סקטור
  DiskInventory.cs       מצאי כוננים (WMI)
  BootSectorAnalyzer.cs  ניתוח סקטור אתחול + גיבוי
  TriageEngine.cs        עץ ההחלטה
  SafeFixes.cs           תיקונים עם ביטול
  Carving/
    FileSignatures.cs    מסד חתימות
    ScanModels.cs        פרויקט/התקדמות + שמירה/טעינה
    ScanEngine.cs        מנוע החיתוך (השהיה/המשך/checkpoint)
MainWindow / DeepScanWindow   ממשק WPF (RTL)
```

## בדיקות
ללא הרשאות מנהל, דרך `dotnet DiskRescue.dll`:
```
--carvetest <out> <report>   בדיקת מנוע החיתוך על קובץ סינתטי
--ntfstest  <out> <report>   בדיקת צינור NTFS מלא (build→parse→recover) על תמונת NTFS סינתטית
```
שתי הבדיקות מאמתות שחזור בייט-אחר-בייט מול תוכן ידוע.
